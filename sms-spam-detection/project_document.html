3. Project Overview

Background / Motivation:
Spam detection is a critical task in modern telecommunications. Unsolicited and often malicious SMS messages, known as spam, negatively impact user experience, can lead to financial fraud, and pose significant security risks. An effective automated system to filter these messages is essential for protecting users and maintaining the integrity of mobile communication services.

Scope:
This project will focus exclusively on the classification of English-language SMS messages into two binary categories: "spam" or "ham" (non-spam). The project will not handle multimedia messages (MMS), messages in other languages, or real-time message interception from a mobile network. The solution will be a standalone model and inference pipeline.

High-level Description:
The project aims to build a machine learning model capable of accurately classifying SMS messages. The system will be developed as a complete pipeline, starting from raw text data, undergoing preprocessing and feature extraction, and finally being fed into a trained classification model to produce a prediction. A simple command-line interface (CLI) will be developed for user interaction.

Dataset / Data Source Summary:
The project will utilize the publicly available "SMS Spam Collection Data Set" from the UCI Machine Learning Repository. This dataset contains 5,574 SMS messages tagged as either "ham" or "spam", providing a substantial foundation for training and evaluating the model.

4. Objectives & Problem Statement

4.1 Problem Statement

The core problem to be solved is the binary classification of SMS messages. Given the raw text content of an SMS message, the system must accurately predict whether it belongs to the "spam" or "ham" class. Key challenges include the short and informal nature of SMS text, which often contains slang, abbreviations, and typos, as well as the inherent class imbalance typically found in spam datasets (where "ham" messages are more frequent).

4.2 Objectives

The specific goals for this project are:

  • To build a predictive machine learning model that achieves high precision and recall for the "spam" class, minimizing both false positives and false negatives.
  • To experiment with and compare the performance of at least two different machine learning algorithms (e.g., Naive Bayes, Logistic Regression).
  • To develop and implement an effective text preprocessing pipeline to clean and normalize SMS data.
  • To create a functional inference pipeline that can classify new, unseen SMS messages.
  • To rigorously evaluate and compare model performance using standard metrics such as accuracy, precision, recall, F1-score, and a confusion matrix.
  • To document the entire process, findings, and suggest potential areas for improvement.

5. Proposed Solution

Approach / Methodology:
The project will employ a supervised learning approach. The text data will be converted into numerical features using techniques like TF-IDF (Term Frequency-Inverse Document Frequency). Several classification algorithms will be implemented and compared, starting with Naive Bayes as a strong baseline, followed by Logistic Regression for comparison.

Pipeline Overview:
The system will follow a clear, sequential pipeline:

  1. Raw SMS Input: The system accepts a raw text string.
  2. Preprocessing: The text is cleaned (lowercasing, punctuation removal), tokenized, and stripped of stopwords.
  3. Feature Extraction: The cleaned text is transformed into a numerical vector using a pre-fitted TF-IDF Vectorizer.
  4. Model Prediction: The numerical vector is fed into the trained classification model.
  5. Prediction Output: The model outputs a prediction ("spam" or "ham") along with a confidence score.

Justification of Choices:

  • Preprocessing: Lowercasing and removing punctuation standardizes the text. Stopword removal helps the model focus on meaningful words.
  • Feature Extraction (TF-IDF): TF-IDF is chosen over a simple bag-of-words (CountVectorizer) because it weighs words by their importance, giving higher scores to words that are frequent in a message but rare across all messages, which is often indicative of spam.
  • Models (Naive Bayes, Logistic Regression): Naive Bayes is a classic, fast, and effective algorithm for text classification. Logistic Regression is a robust, linear model that provides good performance and interpretability, making it an excellent candidate for comparison.

8. System Architecture

Architecture Diagram (Block Diagram)

[Input: SMS Text] -> [Preprocessing Module] -> [Feature Extraction (TF-IDF)] -> [Trained Model] -> [Output: Prediction & Score]

Component Descriptions:

  • Input: The entry point for raw SMS text, either from the command line or a file.
  • Preprocessing Module: Responsible for all text cleaning operations: converting to lowercase, removing punctuation and numbers, tokenization, and stopword removal.
  • Feature Extraction: This module uses a pre-fitted TF-IDF Vectorizer to transform the cleaned text tokens into a numerical feature vector.
  • Trained Model: The core classification algorithm (e.g., Logistic Regression) that takes the feature vector and computes the probability of each class.
  • Output: Formats the final result, displaying the predicted class ("spam" or "ham") and the associated confidence score to the user.

Data Flow:

During Training: Raw data from a CSV file is loaded into pandas. It flows through the Preprocessing and Feature Extraction modules. The resulting feature matrix and labels are split into training and testing sets. The training set is used to train the model.

During Inference: A new SMS message is taken as input. It flows through the same Preprocessing and Feature Extraction modules (using the saved vectorizer). The resulting vector is passed to the loaded, trained model, which generates a prediction. The prediction is then displayed as output.

9. Implementation Steps

  1. Data Acquisition / Loading: Loaded the "SMS Spam Collection Data Set" from a CSV file into a pandas DataFrame.
  2. Exploratory Data Analysis (EDA): Analyzed the distribution of "spam" vs. "ham" messages to understand class imbalance. Examined message length statistics for both classes.
  3. Text Preprocessing: Defined a function to perform the following: convert text to lowercase, remove punctuation and special characters using regex, tokenize the text, and remove English stopwords using the NLTK library.
  4. Feature Extraction: Initialized and applied TfidfVectorizer from scikit-learn to the preprocessed text data to convert it into a numerical matrix.
  5. Train-Test Split: Split the feature matrix and labels into training (80%) and testing (20%) sets using train_test_split.
  6. Model Training: Trained a MultinomialNB model as the baseline. Trained a LogisticRegression model for comparison.
  7. Hyperparameter Tuning: Used GridSearchCV to tune the C parameter for the Logistic Regression model to find the optimal balance between bias and variance.
  8. Model Evaluation: Evaluated both models on the test set. Generated a classification report (precision, recall, F1-score) and a confusion matrix for each. The Logistic Regression model showed superior performance.
  9. Error Analysis: Manually reviewed several misclassified messages from the test set to identify common patterns or failure cases (e.g., very short, ambiguous messages).
  10. Save Model + Vectorizer: Used joblib.dump to save the final, tuned Logistic Regression model and the fitted TfidfVectorizer to disk for later use in inference.
  11. Build Inference Module / Script: Created a Python script (predict.py) that loads the saved model and vectorizer, takes a message as a command-line argument, processes it, and outputs the prediction.
  12. Testing on New Unseen Messages: Tested the inference script with a variety of custom-written spam and ham messages to verify its real-world performance.
  13. Documentation and Reporting: Compiled all findings, code, and results into this final project document.

11. Advantages

  • Automation: The system fully automates the process of spam detection, saving users significant time and effort.
  • High Performance: The Logistic Regression model achieved high precision (0.97) and recall (0.92) for spam, ensuring reliable filtering.
  • Fast and Lightweight: Inference is extremely fast, and the model and vectorizer files are small in size, requiring minimal computational resources.
  • Extensible: The modular pipeline design makes it easy to incorporate new preprocessing steps, try different feature extraction methods, or swap out the classification model.
  • Interpretability: By examining the TF-IDF features and model coefficients, it is possible to understand why a certain message was classified as spam.
  • Usable Interface: The command-line interface provides a simple and direct way to use the model without any complex setup.

13. Conclusion

This project successfully addressed the problem of SMS spam classification by building a robust machine learning pipeline. We started with a clear problem statement and set of objectives, which guided the implementation. The proposed solution, utilizing TF-IDF feature extraction and a Logistic Regression classifier, proved to be highly effective, achieving over 98% accuracy on the test set.

The key to success was effective text preprocessing and the choice of a powerful yet interpretable model. The main challenge faced was the inherent class imbalance, which was managed by focusing on precision and recall metrics for the minority "spam" class. A limitation of the current system is its restriction to English text and its reliance on classical machine learning methods.

In conclusion, the project demonstrates a practical and effective solution to SMS spam detection. The next steps would be to implement the future enhancements, particularly exploring deep learning models and deploying the system as an API to broaden its utility.
